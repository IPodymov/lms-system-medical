# Архитектура

Модульный Django-монолит. `accounts` хранит идентичность, `organizations` — структуру и роли, `courses` — шаблоны и материалы, `learning` — записи и прогресс, `assessments` — тестирование, `grading`, `notifications`, `audit` — самостоятельные ограниченные контексты. Изменения состояния выполняются сервисами, HTTP-слой остаётся тонким.
