# Архитектура

Модульный Django-монолит. `accounts` хранит идентичность, роли и защищённую документацию администратора; `organizations` — структуру и роли; `courses` — шаблоны, конструктор программы, темы и материалы; `learning` — записи и прогресс; `assessments` — тестирование; `grading`, `notifications`, `audit` — самостоятельные ограниченные контексты. Изменения состояния выполняются сервисами, HTTP-слой остаётся тонким.

## Жизненный цикл курса

После первой формы создаётся `Course` со статусом `draft` и автором-владельцем. Автор собирает программу из `CourseSection` → `Lesson` → `ContentBlock`. Текст, файл и тест сохраняются отдельными моделями содержимого; внешние дополнительные материалы курса — в `CourseMaterialLink`.

Черновики выводятся в каталоге только пользователю из `CourseAuthor`. При первой публикации система создаёт для курса активный основной `CourseRun`, открывает период записи на год и назначает автора преподавателем потока. Общий каталог получает только активные потоки опубликованных курсов. Это исключает доступ студентов к незавершённому содержимому.

## Закрытая документация

Ссылка на документацию создаётся в панели системного администратора. В URL передаётся криптографически подписанный Django-токен с идентификатором текущего пользователя; токен действует 24 часа. Обработчик дополнительно проверяет `is_superuser` и совпадение идентификатора в подписи, поэтому ссылка не даёт доступ другой учётной записи.
